# 基本数据类型大小对比分析

## 1. 我的运行环境
- 操作系统：Windows 11 64位
- 编译器：MinGW-w64 (GCC 13.2.0)
- 数据模型：LLP64

## 2. 实测大小（单位：字节）
| 类型 | 我的结果 (MinGW) | Linux/GCC 典型值 | 备注 |
|------|-----------------|------------------|------|
| char | 1 | 1 | C 标准保证永远为 1 |
| short | 2 | 2 | |
| int | 4 | 4 | |
| long | 4 | **8** | ⚠️ 跨平台差异最大 |
| long long | 8 | 8 | |
| float | 4 | 4 | |
| double | 8 | 8 | |
| long double | 16 (或12) | 16 | 取决于编译器实现 |
| 指针 (int*) | 8 | 8 | 64 位系统 |

## 3. 关键发现
- Windows/MinGW 下 **long 只有 4 字节**，而 Linux 下通常为 8 字节。
- 这是因为 Windows 采用 **LLP64** 模型（long 保持 32 位），Linux 采用 **LP64** 模型（long 为 64 位）。
- 两种模型下 int 都是 4 字节，指针都是 8 字节（64位系统）。

## 4. 对嵌入式开发的启示
- 不要直接假设 `long` 或 `int` 的大小，跨平台时使用 `<stdint.h>` 中的固定宽度类型（如 `int32_t`、`uint64_t`）。
- 存储寄存器地址时用 `uintptr_t` 或指针类型，不要用 `long` 强转。
- 写通信协议解析代码时，必须明确每个字段的字节数，使用 `int16_t`、`uint8_t` 等更安全。
